# get-your-offer Firefox Addon

Firefox WebExtension для скрейпинга вакансий с Get on Board, HeadHunter, Habr Career, Gallito, Lever, Jobspresso, Computrabajo, LinkedIn и Wellfound с загрузкой обогащённых данных в Google Sheets (лист Stage).

## Сборка

### Для LibreWolf (рекомендуется)

```bash
python3 build-python.py
```

### Для Firefox

```bash
./build.sh
```

или

```bash
npm run build
```

XPI файл будет создан в `dist/hrscrape2mart.xpi`

**Примечание:** Если у вас проблемы с установкой в LibreWolf, используйте Python-версию сборки (см. [LIBREWOLF.md](LIBREWOLF.md))

### Автоматическая пересборка при изменениях

```bash
./watch.sh
```

или

```bash
npm run watch
```

Скрипт будет отслеживать изменения в файлах и автоматически пересобирать XPI.

Подробнее см. [BUILD.md](BUILD.md)

## Установка

### Первая установка

1. Соберите XPI: `./build.sh` или `npm run build`
2. Откройте Firefox
3. Перейдите в `about:debugging`
4. Нажмите "This Firefox"
5. Нажмите "Load Temporary Add-on"
6. Выберите файл `dist/hrscrape2mart.xpi`

### После изменений кода

**Важно:** После любого изменения кода аддона необходимо пересобрать XPI!

**Правило (обязательно):** если изменён любой файл в `firefox-addon/`, задача считается незавершённой, пока не выполнена сборка `./build.sh` и не получен свежий `dist/hrscrape2mart.xpi`.

1. Запустите автоматическую пересборку: `./watch.sh` (будет пересобирать при каждом изменении)

2. Пересоберите вручную: `./build.sh`

4. Найдите ваше расширение в списке

## Использование

1. Откройте Google Sheet `get-your-offer` (можно оставить вкладку открытой).
2. В листе `Settings` укажите `WebAppUrl` (URL развернутого Apps Script Web App).
3. В листе `ScrapeList` заполните колонки `ScrapePageUrl` и `ScrapePageName` (можно указывать несколько страниц на один `ScrapePageName`).
4. Нажмите иконку расширения и выберите "Scrape All New Jobs".
5. Аддон соберёт вакансии со всех страниц, обогатит карточки и добавит строки в `Stage` (append). В `DataFunnel` будут проставлены `Scraping` → `Staged`.
6. После этого в таблице можно запустить `Validate Stage` и `Increment Load`.
7. Для LinkedIn на текущей странице доступна кнопка `LinkedIn: Hide Viewed (page)` в popup: она скроллит список до конца текущей страницы (без пагинации), кликает крестик у карточек со статусом `Viewed` и использует случайные паузы между кликами.

### Autofill из контекстного меню

1. На любом сайте кликните правой кнопкой в поле ввода (`input`/`textarea`/`contenteditable`).
2. Выберите `get-your-offer Fill` и нужный шаблон.
3. Текст печатается посимвольно в выбранное поле (режим replace).

Текущее поведение:
- поддерживаются поля внутри `iframe` и в `shadow DOM`;
- если шаблон не вставился, в popup в секции Autofill показывается `Last fill failed: ...` (последняя диагностическая ошибка);
- также помогает `Reload` расширения в `about:debugging` и обновление вкладки сайта.

## Примечания

- Обогащение открывает страницы вакансий в фоновых вкладках.
- В `Stage` добавляются только обогащённые вакансии.
- Для LinkedIn добавлена защита от shell-страниц (`0 notifications`): при повторном shell после ретраев x5 и x10 запись не загружается в `Stage`, а учитывается в статусе как `отброшено(shell)`.
- Для LinkedIn на detail-странице при enrich сохраняется `JobApplyUrl` из кнопки/ссылки `Apply` (включая decode `linkedin.com/redir/redirect?...url=...`).
- Для LinkedIn листинг использует случайный джиттер между кликами `Next` (разные интервалы), чтобы поведение пагинации было менее шаблонным.
- В режиме `Scrape All` для LinkedIn порядок обработки страницы: `scroll -> collect urls -> dismiss Viewed -> paginate`.
- Для LinkedIn перед сбором ссылок и перед шагом `dismiss Viewed` список в `.scaffold-layout__list` (или fallback-контейнер) прокручивается вниз до появления ~25 карточек; остановка по отсутствию прироста.
- Для `dismiss Viewed` добавлен таймаут (по умолчанию 9 секунд), чтобы шаг не зацикливался.
- Пагинация LinkedIn завершается, если достигнут лимит 40 страниц или недоступна кнопка `Next`.
- Для Work at a Startup скролл списка компаний теперь идёт до появления новых строк и останавливается после 3 скроллов подряд без прироста.
- Для Work at a Startup перед скроллом добавлено ожидание готовности листа компаний (до 15 секунд): если ссылки компаний не появились, скролл не запускается.
- Для Work at a Startup, если лист не успел прогрузиться (пусто после ready-timeout), выполняется до 3 перезагрузок list-страницы; на ретраях timeout ожидания списка увеличивается в 3 раза.
- Для Torc detail enrichment добавлено отдельное ожидание готовности SPA-деталей (poll `isDetailReady`, увеличенный timeout до 45с), чтобы вкладка не закрывалась до реальной загрузки контента вакансии.
- Используйте "Clear Data" для сброса локального состояния расширения.
